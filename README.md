# クラウドワークス案件収集・管理ツール

## 1. アプリ概要

クラウドワークスの案件探しを効率化するための、案件収集・登録・一覧管理・AI分析ツールです。

「AIで対応できそうな案件」を中心にしつつ、それ以外の幅広いジャンルの案件にも対応できるよう、
まずは応募候補となる案件数を効率よく増やすことを目的としています。

- **第1段階**: 案件の収集・登録・一覧管理（完了）
- **第2段階**: ルールベース一次判定・危険案件検出・AIによる案件適合度分析・スキルプロフィール管理（完了）
- **第3段階（今回追加）**: 案件専用の営業文自動生成・ポートフォリオ自動選択・応募金額/納期提案・
  応募前確認画面（完了）

クラウドワークスへの自動応募（自動送信・ブラウザ自動操作・応募ボタンの自動クリック）は実装していません。
営業文の生成・編集・コピーまでを支援し、実際の応募はユーザーが手動で行う設計です
（詳細は「9. 今後追加したい機能」参照）。

## 3. 実装した機能

### 第1段階（案件収集・管理）

- 案件の手動入力登録（案件本文からの予算・応募期限などの自動抽出付き）
- CSVファイルによる案件の一括登録（列名自動マッピング・手動マッピング対応）
- 公開ページURLからの案件取得（利用規約・robots.txtで許可される範囲に限定した安全設計）
- 案件の重複防止・自動更新（外部ID→URL→タイトル+クライアント名→本文類似度の順で判定）
- 案件一覧の検索・絞り込み・並べ替え、ステータス管理、お気に入り・メモ編集
- ダッシュボードによる各種集計・グラフ表示
- 検索キーワード・除外キーワードなどの設定管理
- CSV・Excel形式での書き出し（文字化け対策済み）
- エラー処理・ログ保存

### 第2段階（AI案件分析。今回追加）

- **ルールベース一次判定**: スキル一致・制作実績一致・対応困難条件・予算・納期・応募人数・
  クライアント評価・本人確認・本文の具体性などから0〜100点のスコアを算出
- **危険・低品質案件の検出**: 外部LINE誘導、初期費用要求、無報酬テストなどをキーワードベースで検出し、
  AIの文脈判定と組み合わせて安全度・危険レベルを判定
- **AI案件適合度判定**: OpenAI / Anthropic Claude / Google Gemini のいずれか（未設定の場合はルールベースのみ）
  を使い、適合度・難易度・必要スキル・不足スキル・予想作業時間・予算評価・応募前の注意点などをJSON構造で分析
- **総合スコア・応募優先度の自動算出**: AI適合度・ルールベース・安全度・予算・納期・応募人数・
  クライアント信頼度・ポートフォリオ一致度を重み付けした総合スコアと、5段階の応募優先度を算出
- **単体分析・一括分析**: 案件を1件ずつ、または条件を指定してまとめて分析
- **分析結果の保存・一覧表示・絞り込み・並べ替え・再分析**
- **スキルプロフィール管理**: 基本情報・スキル・制作実績・希望条件・対応が難しい条件を画面から編集可能
- **AI分析設定画面**: プロバイダー・モデル・スコアの重み・優先度境界値・危険キーワード等を画面から調整
- **分析結果のキャッシュ**: 同一案件・同一プロフィールでの無駄なAPI呼び出しを防止

### 第3段階（営業文自動生成。今回追加）

- **案件専用の営業文自動生成**: AI分析結果・スキルプロフィール・制作実績をもとに、案件ごとに内容の異なる
  営業文をJSON構造で生成（OpenAI / Anthropic / Gemini。営業文生成用モデルは案件分析用と別設定可能）
- **ポートフォリオ自動選択**: 案件がデザイン系／AI・開発系／AI×デザイン複合系のどれに近いかを判定し、
  関連度0〜100点で1〜3件のポートフォリオを自動選択（デザイン案件ではforiioを、開発案件ではAI・開発
  ポートフォリオ／GitHubを優先。手動での追加・削除・並べ替えも可能）
- **応募金額の提案**: 案件の予算・募集形式・予想作業時間・カテゴリから、最低受注金額を下回らない範囲で
  金額を提案（画面から基準単価等を編集可能）
- **納期の提案**: 予想作業時間・1日あたりの稼働時間・デザイン案の作成/修正/素材待ち/API審査待ちなどの
  バッファを考慮した最短/安全/推奨納期を算出し、対応が難しい場合は断定せず警告を表示
- **クライアント質問の自動抽出と回答案作成**: 案件本文から応募者への質問を検出し、登録済み実績の範囲内で
  回答案を作成（関連するURLのみを使用し、無関係なURLは挿入しない）
- **危険・低品質案件での生成停止**: 危険レベルcritical・安全度が基準未満・本文がほぼ空・購入/初期費用要求
  等の案件では営業文を自動生成せず、理由付きで警告を表示
- **AI出力の安全性検証**: 登録されていないURLの自動削除、結果保証表現（「必ず売上が上がる」等）の検出、
  実案件経験が未登録の状態での「実務経験あり」断定の検出
- **営業文の編集・再生成・バージョン管理**: 直接編集、定型指示（短くする/丁寧にする/実績を強調する 等）
  による再生成、編集履歴の保存と過去バージョンへの復元
- **AI APIなしでのテンプレート生成**: 19種類の案件カテゴリ別テンプレートにより、AI未使用でも下書きを作成
- **応募前確認画面・応募準備ステータス管理**: 案件情報・分析結果・営業文・チェックリストを1画面で確認し、
  10段階の応募準備ステータス（未作成〜応募済み）を管理
- **営業文のコピー機能**: 全文/短縮版/金額/納期/各種URL/応募内容全体を個別にコピー可能
- **営業文生成結果のキャッシュ**: 案件内容・分析結果・選択ポートフォリオ・生成条件からハッシュを作成し、
  手動編集した内容は再生成（force_regenerate未指定時）で上書きしない

## 4. 使用技術

- Python 3.11以上（開発・動作確認は Python 3.14 で実施）
- Streamlit（画面）
- pandas（データ処理）
- SQLite（データベース）
- requests / BeautifulSoup4 / lxml（URL取得・AI API呼び出し）
- python-dotenv（環境変数管理）
- pytest（自動テスト。AI APIはモックを使用し、有料APIは一切呼び出さない）
- pytz / python-dateutil（日時処理）
- pydantic（データモデル定義・AIレスポンスの構造検証）
- tenacity（取得処理のリトライ制御）
- validators（URL検証）
- openpyxl（Excel書き出し）
- Altair（グラフ表示）

AIプロバイダー（OpenAI / Anthropic / Gemini）は、いずれも専用SDKを追加せず、
既存の `requests` ライブラリで直接HTTP呼び出しする構成にしています。
理由は「8. 工夫した点」参照。

第3段階（営業文自動生成）でも新しい外部パッケージは追加していません。
案件分析用のAIクライアント・JSON抽出ロジック（`src/ai/`）をそのまま再利用し、
営業文生成専用のプロンプト・スキーマ（`src/application/`）のみを追加しています。

## 5. 機能一覧

### 5-1. ダッシュボード

保存案件数・本日取得件数・未確認/確認済み/応募候補/見送り/応募済みの各件数をカード表示し、
直近7日間の取得数推移、募集形式別・検索キーワード別・ステータス別の件数をグラフ／表で確認できます。

### 5-2. 案件を追加（3方式）

| 方式 | 内容 |
|---|---|
| URLから取得 | 許可されたドメインの公開ページのみ取得可（下記「利用規約と自動取得に関する注意事項」を参照） |
| 手動入力 | 案件タイトルのみ必須。本文貼り付けで予算等を自動抽出し、保存前に編集可能 |
| CSVアップロード | 列名自動マッピング、プレビュー表示、登録結果（新規/更新/重複/エラー）を表示 |

#### CSVの形式

以下の列名（英語・日本語どちらも可）に対応しています。列名が異なる場合は画面上で手動マッピングできます。

```
title / 案件タイトル（必須）
url / 案件URL
description / 案件概要
body / 案件本文
job_type / 募集形式
category / カテゴリ
budget / 予算
budget_min / 予算下限
budget_max / 予算上限
published_at / 掲載日時
deadline / 応募期限
applicant_count / 応募人数
recruitment_count / 採用人数
client_name / クライアント名
client_rating / クライアント評価
identity_verified / 本人確認
external_job_id / 外部案件ID
keyword / 検索キーワード
memo / メモ
```

サンプルCSVは `data/sample_jobs.csv` にあります（「案件を追加」→「CSVアップロード」タブからもダウンロード可能）。
文字コードは UTF-8(BOM付き) と Shift-JIS(CP932) に対応しています。

### 5-3. 案件一覧

保存済み案件の検索・絞り込み（フリーワード／キーワード／予算／募集形式／カテゴリ／期限／評価／ステータス等）、
並べ替え（新着順／取得日時順／期限が近い順／予算が高い順／応募人数が少ない順／評価が高い順）、
ステータスの個別・一括変更、お気に入り・メモ編集、CSV/Excel書き出しに対応しています。
第2段階の追加として、**総合スコア・AI適合度・安全度・応募優先度・難易度・危険レベル・分析状況・分析日時**
の列も表示されます（未分析の案件は「未分析」と表示されます）。
第3段階の追加として、**営業文作成状況・応募準備ステータス・提案金額・提案納期・コピー済みかどうか・
営業文最終更新日時**の列も表示され、案件詳細から「営業文を生成する」「AI・開発ポートフォリオを優先選択」
「デザインポートフォリオを優先選択」「コピー済みにする」「営業文一覧/応募前確認画面を開く」を実行できます。

### 5-4. 設定

検索キーワード・除外キーワードの追加/削除、最低予算、最大取得件数、デフォルト募集形式、
取得時の待機秒数、1日の将来の応募上限数、タイムゾーンを設定・保存できます（SQLiteのsettingsテーブルに保存）。

### 5-5. AI案件分析（第2段階で追加）

未分析件数・分析済み件数を表示し、以下から分析対象を選んで実行できます。

- 未分析案件のみ / 選択した案件 / 応募候補ステータスのみ / 本日取得した案件 / 条件に合う案件 / 全案件を再分析

一括分析件数の上限、分析間の待機秒数、最低限必要な本文文字数を指定でき、
「ルールベースのみで実行する」「強制再分析する（キャッシュ無視）」も選択できます。
実行後は成功・失敗・スキップ・API使用・ルールベースのみ・キャッシュ利用の各件数を表示します。

### 5-6. 分析結果一覧（第2段階で追加）

分析済み案件を一覧表示し、総合スコア・AI適合度・安全度・応募優先度・応募推奨度・難易度・危険レベル・
予算評価・分析状況などで絞り込み、8種類の基準で並べ替えできます。
案件を選択すると、要約・必要スキル・一致/不足スキル・関連ポートフォリオ・予想作業時間・予算評価・
応募時の強み・注意点・応募前に確認すべき質問・応募方針・AIの判定理由・安全性分析の詳細を確認できます。
「再分析」「ルールベースのみ再計算」「応募候補に変更」「見送りに変更」「判定結果をコピー（JSON）」
「案件ページを開く」のボタンを用意しています。

### 5-7. スキルプロフィール（第2段階で追加、第3段階でポートフォリオ管理を拡張）

基本情報、スキル（カテゴリ・習熟度・経験区分）、希望条件、対応可能な業務、対応が難しい業務、
除外条件を画面から追加・編集・削除できます。ここで登録した内容がAI分析・営業文生成の基礎情報になります。

「制作実績」タブは第3段階でポートフォリオ管理画面として拡張し、種類（AI・開発／デザイン／総合）、
対象案件カテゴリ、デザインツール、営業文用の紹介文、優先度、開発/デザイン/AI×デザイン案件向けフラグ、
公開・非公開、表示順を編集できます。「AI・開発」「デザイン」「総合」「非公開」でのフィルター表示にも対応しています。

### 5-8. AI分析設定（第2段階で追加）

AIプロバイダー・モデル名・APIタイムアウト・最大再試行回数・一括分析上限・分析間の待機秒数・
使用トークン上限・1日あたりの分析上限件数・総合スコアの重み・応募優先度の境界値・危険キーワード
（カテゴリ別）・加点/減点キーワード・分析対象の絞り込み条件（最低予算・最大応募人数・最低クライアント評価・
本人確認必須）・ルールベースのみで動作する設定を保存できます。APIキー自体はこの画面には保存されません
（「10. APIキーの設定方法」参照）。

### 5-9. 営業文生成（第3段階で追加）

案件を選択し、営業文タイプ（トーン・全17種類）・文章の長さ・見出しの有無・紹介するポートフォリオ・
追加で伝えたい内容/含めたくない内容・応募金額/納期の指定（任意）を指定して営業文を生成できます。
案件カテゴリからおすすめのトーンを自動選択し、AI APIを使わずテンプレートのみで生成することも可能です。
一括生成タブでは、未生成案件のみ／選択した案件／応募候補ステータスのみ／応募優先度が高い案件のみ／
全案件を再生成、から対象を選んでまとめて生成できます。「料金・納期の設定」タブから、基準時間単価・
最低受注金額・標準修正回数・1日あたりの稼働時間・各種バッファ日数を編集できます。

### 5-10. 営業文一覧（第3段階で追加）

生成済み営業文を応募準備ステータス・生成状況で絞り込んで一覧表示し、案件を選んで詳細を確認できます。
営業文の直接編集、定型指示（短くする/詳しくする/丁寧にする/熱意を強くする/技術説明を増やす/
デザイン説明を増やす/実績を強調する/提案内容を増やす/初心者らしい誠実さを加える/不自然な表現を修正する/
誤字脱字を修正する）による再生成、全文・短縮版・応募内容全体（JSON）のコピー、応募準備完了/応募済み/
見送りへのステータス変更、編集履歴の確認と過去バージョンへの復元ができます。

### 5-11. 応募前確認（第3段階で追加）

案件情報・AI分析結果（総合スコア・優先度・危険レベル・一致/不足スキル・注意点）・応募内容
（営業文・選択ポートフォリオ・金額・納期・クライアント質問への回答・確認事項）・ユーザー用メモを
1画面にまとめて表示し、10項目のチェックリストを管理します。すべてチェックすると応募準備ステータスが
自動的に「応募準備完了」に変わります。

### データベースの保存場所

`data/jobs.db`（SQLite）に保存されます。アプリ初回起動時に自動でテーブルが作成・マイグレーションされます。
このファイルは個人の収集データを含むため `.gitignore` でGit管理対象外にしています。

## 6. 実装手順

### 環境構築方法

```bash
# 1. リポジトリのクローン、またはこのフォルダに移動
cd crowdworks-sales-assistant

# 2. 仮想環境の作成
python -m venv venv

# 3. 仮想環境の有効化
# Windows (PowerShell)
venv\Scripts\Activate.ps1
# Windows (コマンドプロンプト)
venv\Scripts\activate.bat
# macOS / Linux
source venv/bin/activate

# 4. パッケージのインストール
pip install -r requirements.txt

# 5. 環境変数ファイルの作成（AI分析機能を使わない場合は編集不要）
cp .env.example .env
```

### アプリの起動方法

```bash
streamlit run app.py
```

ブラウザが自動的に開かない場合は、表示された `Local URL`（例: http://localhost:8501）にアクセスしてください。
初回起動時に `data/jobs.db` が自動生成され、必要なテーブルが作成されます
（第2段階のAI分析用テーブルも自動でマイグレーションされ、初期スキルプロフィールが投入されます）。

サンプルデータを試しに登録したい場合は、以下を実行してください（`data/sample_jobs.csv` を取り込みます）。

```bash
python -m src.seed
```

### テストの実行方法

```bash
pytest
```

190件のテスト（第1段階: データベース初期化、案件の新規登録・更新・重複判定、URL正規化、予算/日付の解析、
CSV列名マッピング、CSVインポート、入力検証、ステータス変更、ユーザー入力項目の保持／
第2段階: ルールベース分析、危険案件検出、総合スコア計算、AIレスポンス解析、マイグレーション、
分析結果キャッシュ、AIクライアント（モック使用）／
第3段階: ポートフォリオ自動選択、応募金額・納期提案、クライアント質問抽出、営業文テンプレート生成、
営業文AIレスポンス解析、営業文生成キャッシュ、営業文生成の停止条件・安全性検証、DBマイグレーション、
営業文サービス（バージョン管理・チェックリスト・手動編集の保持・一括生成。AI APIはモック使用））が、
すべて成功することを確認済みです。

## 8. 工夫した点

### 利用規約と自動取得に関する注意事項

実装前に、クラウドワークスの `https://crowdworks.jp/robots.txt` を確認しました。結果は以下の通りです。

- **`ClaudeBot`（Anthropic）・`GPTBot`（OpenAI）・`meta-externalagent`（Meta）など、AIクローラーを名指しで完全にブロック**しています。
- `/api/` 配下は `/api/v3/public/` を除き Disallow（クロール禁止）です。
- `/admin/`, `/internal/`, `/attachments/` なども Disallow です。
- 利用規約ページ（`/about/terms_of_service` 等）へのアクセスは確認できませんでした（404）。

このように、サイト運営者がAIエージェントによる自動アクセスを明示的に拒否する意思を示しているため、
**本ツールから crowdworks.jp への自動スクレイピングは一切実装していません。**
第2段階のAI分析機能も、あくまで**利用者自身が手動入力・CSVで登録した案件情報を分析するだけ**であり、
クラウドワークスへの追加アクセスは発生しません。

URL取得機能（`src/job_collector.py`）は、`src/validators.py` の `is_blocked_domain` /
`validate_fetch_url` によって **crowdworks.jp ドメインへのアクセスをアプリ側で強制的にブロック**しています。

### 分析結果を別テーブルに分離した設計

要件では `jobs` テーブルへ約30列を追加する構成と、分析結果を別テーブル（`job_analyses`）に分ける構成の
どちらでもよいとされていたため、**既存の `jobs` テーブルには一切手を加えず、`job_analyses` を新設する方式**
を選びました。理由は、第1段階の既存コード（`repositories.py` の `JOB_COLUMNS` や重複判定ロジックなど）を
一切変更せずに済み、既存機能を壊すリスクを最小化できるためです。分析結果は履歴として複数回分保存され、
案件一覧・分析結果一覧では「最新の分析結果」を `LEFT JOIN` で結合して表示します。

### AIプロバイダー共通インターフェースをrequestsで統一

`BaseAIClient` を継承した `OpenAIClient` / `AnthropicClient` / `GeminiClient` は、
いずれも公式SDKを使わず `requests` で直接HTTP呼び出しする構成にしています。
理由は、3社のSDKを同時に追加するとバージョン依存関係が複雑化しやすく、
「使用していないライブラリを無理に追加しない」「バージョン競合が起きない構成にする」という要件を
満たしやすいためです。また、この構成によりテストでは `requests.post` をモックするだけで
実際の有料APIを一切呼び出さずに全プロバイダーの挙動（正常応答・タイムアウト・認証エラー・
レート制限・不正レスポンス）を検証できています。

### AI分析結果のJSON解析とフォールバック

AIの応答は「1. コードブロックの除去→2. 前後の文章を除いた抽出→3. 末尾カンマ等の軽微な修正→
4. 1回だけ再生成→5. すべて失敗したらルールベース結果を使用」という順序で復旧を試みます
（`src/ai/response_parser.py`, `src/analysis/job_analyzer.py`）。
スコアは pydantic のバリデータで 0〜100 にクランプし、不正な列挙値は安全な既定値へフォールバックするため、
AIが多少壊れた応答を返しても、アプリやDB保存処理が落ちることはありません。

### AI APIを使わない場合の代替判定

AIプロバイダーが未設定（`.env` にキーがない、または設定画面で「ルールベースのみ」を選択）の場合でも、
ルールベーススコア・キーワードベースの危険度判定だけで総合スコア・応募優先度・安全度を算出します。
AIを使った場合との違いは、AI適合度スコアの代わりにルールベーススコアを使うことと、
案件要約・必要スキル等のAI由来の項目が空になることのみで、画面の表示や操作は同じです。

### 実績の過大評価を防ぐプロンプト設計

ユーザーのスキルは「学習」「個人開発」「公開実績」「実案件」「その他」の経験区分で管理し、
初期プロフィールでは制作実績に直接紐づくスキルのみ「公開実績」、それ以外は「個人開発」として登録しています
（`src/profile/default_profile.py`）。AIへのプロンプトでも、実務経験が確認できないスキルを
「実務経験あり」と断定しないこと、案件本文に書かれていない内容を推測しすぎないことを明示的に指示しています。

### キャッシュのバージョン管理

分析結果のキャッシュキーは、案件内容・プロンプト版・AIプロバイダー/モデルに加えて、
プロフィールの `updated_at`（秒単位）と `version`（更新のたびに増加する整数カウンタ）の両方を含めています。
`updated_at` だけだと同一秒内に連続更新した場合にキャッシュが誤って再利用される可能性があったため、
より確実な変更検知のために `version` を追加しました（テストで実際に検出し、修正しています）。

### 営業文本文を独立テーブルに分離し、履歴を残す設計

要件で明示的に禁止されていた「営業文本文を `jobs` テーブルへ1件だけ保存する設計」を避け、
`application_drafts`（現在の下書き）と `application_versions`（編集・再生成のたびのスナップショット）
を分離しました。「直接編集」「定型指示による再生成」「強制再生成」のいずれの操作でも、変更前の状態を
`application_versions` へ記録してから上書きするため、「元の生成文に戻す」機能が安全に実現できます。

### ポートフォリオ自動選択のルール設計

案件本文をキーワード一致でデザイン系／AI・開発系／AI×デザイン複合系に分類し（`portfolio_category_classifier.py`）、
ポートフォリオ側の `for_design` / `for_development` / `for_ai_design` フラグと突き合わせて関連度スコア
（0〜100点）を算出します。デザイン案件ではforiioのデザインポートフォリオへの一致ボーナスが最も大きくなるよう、
AI・開発案件ではAI・開発ポートフォリオ／GitHubへの一致ボーナスが大きくなるよう重みを設定し、
関連度が閾値未満のポートフォリオは無理に選択せず「関連実績なし」として扱います
（虚偽の実績を営業文に書かせないため）。

### AI出力の安全性検証を機械的に行う

AIが生成した営業文に、候補として渡していないURLが含まれていた場合は正規表現で検出し自動的に削除します
（`application_validator.py`）。「効果を保証」「必ず成果」等の結果保証表現、実案件経験が未登録の状態での
「実務経験が豊富」等の断定表現も検出し、警告として営業文一覧画面に表示します。これにより、AIの生成内容を
無条件に信頼せず、機械的にチェック可能な範囲は自動修正・警告する二段構えにしています。

### 営業文生成の停止条件と危険案件判定の連携

第2段階で実装済みの危険キーワード検出・安全度スコアをそのまま再利用し、危険レベルがcritical、
安全度が基準未満、案件本文がほぼ空、教材購入/初期費用要求等のキーワードに該当する場合は、
`GenerationBlockedError` を送出して営業文の生成自体を行いません（一括生成では該当案件をスキップして
処理を継続します）。

### 営業文キャッシュと手動編集の共存

営業文のキャッシュキー（`application_cache.py`）は、案件内容・AI分析結果・プロフィールのバージョン・
選択ポートフォリオ・生成条件（タイプ/トーン/長さ/金額/納期/追加指示）・プロンプト版・プロバイダー/モデルから
算出します。`application_drafts` は案件ごとに1行を更新していく設計のため、同一条件での「再生成」は
DBに保存済みの最新状態（＝手動編集後の内容であればその内容）をそのまま返し、AIを再度呼び出しません。
内容を変えて生成し直したい場合のみ「強制再生成」を選ぶ設計にすることで、
「手動編集した文章を再生成時に上書きしない」という要件を自然に満たしています。

### そのほかの工夫（第1段階から継続）

- 重複判定は「外部ID → 正規化URL → タイトル+クライアント名 → 本文類似度」の4段階で行い、
  更新時もステータス・お気に入り・メモといったユーザー入力は上書きされないようにしています。
- CSVは UTF-8(BOM付き)・Shift-JISの両方に対応し、列名の英語/日本語表記の違いも自動マッピングで吸収します。
- エラーはすべて日本語のメッセージで表示しつつ、詳細情報は `logs/app.log` に記録し、
  1件のエラーでアプリ全体や一括処理全体が止まらないようにしています（行/URL単位で処理継続）。
- 日時はすべて日本時間（Asia/Tokyo）で保存・表示しています。

## 10. 学んだこと

- スクレイピング機能を作る前に robots.txt・利用規約を確認するプロセスの重要性。
  AIエージェント自身が対象になり得る（`ClaudeBot` 拒否など）ことを踏まえ、
  「技術的に取得できるか」ではなく「取得してよいと明確に確認できるか」を基準に設計判断する必要があること。
- 重複防止や「ユーザー入力項目は上書きしない」というルールは、データ収集ツールを繰り返し実行する上で
  非常に重要であり、優先順位付きの判定ロジックとして明示的に設計することでテストしやすくなること。
- AIの構造化出力は「必ず壊れる前提」で設計する必要があること。JSON抽出の多段フォールバックと
  スコアのクランプ処理を用意したことで、実際に不正な列挙値や範囲外スコアのテストケースを
  安全に通過させることができた。
- 時刻ベースのキャッシュキーは秒単位の解像度では不十分な場合があること。
  テストを書いたことで「同一秒内の連続更新でキャッシュが誤って再利用される」不具合を実装中に検出でき、
  バージョンカウンタを追加する設計改善につながった。
- 分析結果を独立テーブルに分離する設計は、既存の安定した機能（案件収集・重複防止等）に触れずに
  大きな機能を追加できる点で、段階的な機能拡張と相性が良いこと。
- AIが生成する自然文（営業文）は、構造化データ（分析JSON）よりも「機械的に安全性を検証しにくい」性質があること。
  URLの正規表現抽出・許可リスト照合や、結果保証表現のキーワード検出のように、テキスト内容そのものへの
  検証ロジックを別途用意する必要があると分かった。
- 「再生成でユーザーの手動編集を上書きしない」という要件は、キャッシュキー設計だけでは解決できず、
  「現在の下書きは1行を更新し続ける」「変更前の状態は必ずバージョンとして退避してから上書きする」という
  データモデル自体の設計判断とセットで初めて安全に実現できること。
- ポートフォリオ自動選択のような「複数条件から関連度を採点して選ぶ」ロジックは、採点式をベタ書きするより
  「キーワード一致・カテゴリ一致・URL有無」のように要素ごとに加点を分解してテストを書いた方が、
  意図通りに動いているかを検証しやすいこと。

## AI案件分析の詳しい使い方

### ルールベース判定の仕組み

基準点50点から、スキル一致・関連制作実績・対応困難条件・除外キーワード・加点/減点キーワード・
予算・納期・応募人数・クライアント評価・本人確認・本文の具体性を加減点し、0〜100点に収めます
（`src/analysis/rule_based_analyzer.py`）。各項目の重みは「AI分析設定」画面から調整できます。

### 危険案件判定の仕組み

案件本文等をカテゴリ別の危険キーワードと単純一致させ、安全度スコア・危険レベルを算出します
（キーワード一致はあくまで一次判定であり、AIが利用可能な場合はAIの文脈判定と統合し、
より安全側（危険度が高い方）を採用します）。単純一致結果とAI判定結果は、
分析結果画面で「キーワード一致」「AI判定」として区別して表示されます。

### 総合スコアの仕組み

AI適合度35% + ルールベース20% + 安全度15% + 予算評価10% + 納期の余裕5% + 応募人数5% +
クライアント信頼度5% + ポートフォリオ一致度5%（合計100%）の重み付き平均で算出します
（AI未使用時はAI適合度の代わりにルールベーススコアを使用）。
重みは「AI分析設定」画面から変更でき、合計が100%にならない場合は保存前に警告し、既定値へフォールバックします。
総合スコアと危険レベルから応募優先度（最優先/優先/応募候補/要確認/見送り候補）を判定し、
危険レベルが高い案件は総合スコアが高くても優先度が強制的に引き下げられます。

### スキルプロフィールの編集方法

「スキルプロフィール」ページの各タブから、基本情報・希望条件・対応が難しい業務・スキル・制作実績を
追加・編集・削除できます。ここで登録した内容がAI分析のプロンプトに使用されます。

### APIキーの設定方法

1. `.env.example` を `.env` にコピーする
2. 使用したいプロバイダーのAPIキーを設定する（例: `OPENAI_API_KEY=sk-...`）
3. `AI_PROVIDER` に `openai` / `anthropic` / `gemini` のいずれかを設定する（または「AI分析設定」画面から変更）
4. アプリを再起動する

APIキーはコードやデータベース・画面上には一切保存されません。「AI分析設定」画面ではキーの
設定有無（設定済み/未設定）のみを表示します。

### AIプロバイダーの変更方法

「AI分析設定」画面の「AIプロバイダー・API設定」から変更し、保存するだけです。
モデル名も同じ画面から変更できます（`.env` の `OPENAI_MODEL` 等は初期値として使用されます）。

### AI APIなしで利用する方法

`.env` に何も設定しない、または「AI分析設定」画面で「AI分析を使わずルールベースのみで動作する」に
チェックを入れると、AI APIを一切呼び出さずにルールベース判定・危険案件検出のみで動作します。
アプリが停止することはありません。

### 一括分析の方法

「AI案件分析」ページで分析対象（未分析案件のみ／選択した案件／応募候補ステータスのみ／
本日取得した案件／条件に合う案件／全案件を再分析）を選び、一括分析件数の上限と待機秒数を確認して
「分析を開始する」を押します。案件間には設定した待機秒数を挟み、APIへの連続送信を避けます。

### 分析結果の見方

「分析結果一覧」ページで、総合スコア・AI適合度・ルールベース・安全度・応募優先度（🌟最優先〜⚪見送り候補）・
危険レベル（🟢〜🔴）などをバッジ表示します。案件を選択すると詳細（要約・必要/一致/不足スキル・
関連ポートフォリオ・予想作業時間・応募前の注意点・AIの判定理由・検出されたリスク等）を確認できます。

### キャッシュの仕組み

案件タイトル・本文・予算・納期・プロフィールの更新状態・プロンプト版・AIプロバイダー/モデルから
ハッシュ値を作り、同一のハッシュを持つ分析結果が既に存在する場合は再分析せずに再利用します。
案件内容またはプロフィールを変更すると自動的に再分析されます。「強制再分析する」を選ぶとキャッシュを無視します。

### API費用を抑える方法

- キャッシュにより同一内容の再分析を防止
- 案件本文は最大4000文字に短縮してから送信（`AI_MAX_BODY_CHARS`）
- 「最低限必要な案件本文文字数」未満の案件はAI分析をスキップしルールベースのみで判定
- 一括分析件数の上限・待機秒数・1日あたりの分析上限件数を設定可能
- 「ルールベースのみで実行する」を選べばAPIを一切使わない

### データベースマイグレーション

`src/migrations/add_job_analysis_tables.py` がアプリ起動時（`init_db()`）に自動実行されます。
`CREATE TABLE IF NOT EXISTS` と、既存テーブルへの不足カラムの追加チェックのみを行うため、
何度実行しても安全で、既存の案件データを削除・変更することはありません。
初期スキルプロフィールは、`user_profiles` に `profile_name='default'` が存在しない場合のみ投入されます。

## 営業文生成の詳しい使い方

### 営業文生成機能

「営業文生成」ページで案件を選び、営業文タイプ（トーン）・文章の長さ・見出しの有無・追加で伝えたい内容・
含めたくない内容を指定して「営業文を生成する」を押します。危険レベルが高い案件などでは自動生成が
停止し、理由が画面に表示されます。生成された営業文は「営業文一覧」ページで確認・編集できます。

### 開発案件とデザイン案件の自動分類

案件本文・タイトル・カテゴリをキーワードで判定し、デザイン系／AI・開発系／AI×デザイン複合系のいずれかに
分類します（`src/portfolio/portfolio_category_classifier.py`）。この分類結果が、ポートフォリオ自動選択・
テンプレートカテゴリ判定・おすすめトーンの自動選択にそのまま使われます。

### AI・開発ポートフォリオ / デザインポートフォリオ

初期登録として、以下の3件が自動投入されます（既に同名の実績がある場合は重複登録しません）。

| ポートフォリオ | URL | 用途 |
|---|---|---|
| AIエンジニア・Web制作ポートフォリオ | https://moedaichi0629-ai.github.io/landing-page/ | AI開発・Webアプリ・API連携・業務自動化・ホームページ制作案件 |
| グラフィック・Webデザイン ポートフォリオ（foriio） | https://www.foriio.com/rilymoe0902 | バナー・SNS投稿画像・ロゴ・名刺・チラシ・Webデザイン案件 |
| GitHub | https://github.com/moedaichi0629-ai | 技術力の確認が必要なAI・開発案件 |

既存7件の制作実績（Googleマップ×ホームページ生成システム、歌舞伎予習AIチャットボット 等）にも、
URLが判明しているものは自動補完し、判明していないものは空欄のまま（虚偽のURLは作成しません）にしています。

### ポートフォリオ自動選択の仕組み

案件との関連度（0〜100点）は、使用技術・キーワードの一致、対象案件カテゴリとの一致、
デザイン/開発/AI×デザイン案件フラグとの一致、公開URL・GitHub URLの有無、優先度設定から算出し、
関連度が高い順に最大3件まで自動選択します（`src/portfolio/portfolio_matcher.py`）。
デザイン案件ではforiioのデザインポートフォリオが、AI・開発案件ではAI・開発ポートフォリオやGitHubが
優先的に選ばれます。関連性が低い場合は無理に選択せず「関連実績なし」となり、案件一覧・スキルプロフィール
ページから手動での追加・削除もできます。

### 応募金額提案の仕組み

募集形式（時間単価制/固定報酬制）、案件カテゴリ、予想作業時間、予算範囲から応募金額を算出します
（`src/pricing/price_calculator.py`）。最低受注金額を下回らないようにし、作業内容が確定していない場合は
「目安」であることを明記します。基準時間単価・最低受注金額などは「営業文生成」ページの
「料金・納期の設定」タブから編集できます。

### 納期提案の仕組み

予想作業時間・1日あたりの稼働時間から必要作業日数を算出し、デザイン案の作成・修正対応・素材待ち・
外部API審査待ちなどのバッファを加算して最短/安全/推奨納期を提案します（`src/delivery/delivery_calculator.py`）。
難易度が高い案件や、応募期限までの残り日数が推奨納期より短い場合は、対応可能と断定せず警告を表示します。

### 営業文の編集・再生成の方法

「営業文一覧」ページで、テキストエリアへの直接編集（再生成では上書きされません）、
定型指示（短くする/丁寧にする/実績を強調する 等）を選んでの再生成、「再生成する」ボタンによる
強制再生成ができます。変更のたびに履歴が保存され、過去のバージョンへワンクリックで復元できます。

### 応募前確認画面の使い方

「応募前確認」ページで案件を選ぶと、案件情報・AI分析結果・応募内容（営業文・ポートフォリオ・金額・納期・
クライアント質問への回答）を1画面で確認できます。10項目のチェックリストをすべてチェックすると、
応募準備ステータスが自動的に「応募準備完了」に変わります。

### テンプレート管理

`application_templates` テーブルに、案件カテゴリ（AI開発・API連携・バナー制作 等、19種類）ごとの
初期テンプレートが自動投入されます。AI APIが利用できない場合や「AI APIを使わずテンプレートのみで生成する」
を選んだ場合は、このテンプレート＋ルールベースのロジックで営業文の下書きを作成します。

### AI APIなしで営業文を作る方法

`.env` にAPIキーを設定しない、または「営業文生成」ページで「AI APIを使わずテンプレートのみで生成する」に
チェックを入れると、AI APIを一切呼び出さずに営業文の下書きを作成します。この場合も、ポートフォリオ自動選択・
応募金額/納期提案・クライアント質問抽出はすべて動作します。

## よくあるエラーと対処方法

| 症状 | 原因 | 対処方法 |
|---|---|---|
| URL取得で「crowdworks.jpは…自動取得することはできません」と表示される | crowdworks.jpはrobots.txtでAIクローラーを禁止しているため、アプリ側でブロックしている | 手動入力またはCSVアップロードで登録してください |
| URL取得で「ログインが必要なページのため取得を中断しました」 | 取得先がログインを要求するページだった | ログイン不要な公開ページのURLを指定してください |
| URL取得で「CAPTCHAが表示されたため取得を中断しました」 | 取得先がCAPTCHA認証を要求した | CAPTCHA回避は行いません。手動入力をご利用ください |
| CSVアップロードで「CSVの文字コードを判定できませんでした」 | UTF-8/Shift-JIS以外の文字コードで保存されている | Excel等で「CSV UTF-8」形式で保存し直してください |
| CSVアップロードで特定行がエラーになる | その行に案件タイトルが入っていない、または値の形式が不正 | 画面に表示される行番号・理由を確認し、CSVを修正してください |
| ダッシュボードが真っ白/エラー表示になる | データベースファイルが破損している可能性 | `logs/app.log` を確認し、必要であれば `data/jobs.db` を削除して再起動（再作成）してください |
| `streamlit run app.py` が起動しない | 仮想環境が有効化されていない、または依存パッケージ未インストール | `venv`を有効化し `pip install -r requirements.txt` を再実行してください |
| 「AI案件分析」で分析を実行してもAIが使われない | APIキー未設定、または「ルールベースのみ」設定が有効 | 「AI分析設定」画面でプロバイダーの状態（設定済み/未設定）を確認し、`.env`にAPIキーを設定してください |
| 分析結果のスコアがすべて同じような値になる | AI未使用でルールベースのみ判定になっている | 「AI案件分析」ページの警告表示、または分析詳細の「分析: rule_only」表示を確認してください |
| 「重みの合計が100%になっていません」と表示される | AI分析設定でスコアの重みを変更し、合計が100%からずれた | 各項目のパーセンテージを調整し、合計を100%にしてから保存してください |
| 分析が途中で止まる/失敗件数が多い | AI APIのレート制限やタイムアウト | 分析間の待機秒数を増やす、一括分析件数を減らす、またはルールベースのみで実行してください |
| 「営業文の自動生成を停止しました」と表示される | 危険レベルcritical・安全度が基準未満・案件本文がほぼ空・購入/初期費用要求等のキーワードを検出した | 表示された理由を確認し、案件内容に問題がなければ「AI案件分析」で再分析するか、内容を精査した上で応募判断してください |
| 営業文に案件と無関係なURLが含まれる/削除される | AIが未登録・無関係なURLを生成し、安全性検証で自動削除された | 「営業文一覧」の警告表示を確認し、正しいポートフォリオが選択されているか見直してください |
| 「関連実績なし」と表示される | 案件内容と一致する制作実績が登録されていない | スキルプロフィールの「制作実績」タブから該当する実績を追加するか、案件一覧から手動でポートフォリオを選択してください |
| 提案金額・納期が「目安」表示になる | 案件本文の情報が少なく、予想作業時間などが確定できない | 応募後にクライアントとすり合わせるか、AI分析を実行してから再度営業文を生成してください |
| 再生成しても手動編集した内容が変わらない | キャッシュにより、内容が同じ条件では再生成をスキップしている（意図した動作） | 内容を変えたい場合は「強制再生成する」にチェックを入れて再生成してください |

## GitHubへ保存する方法

このフォルダは独立したツールとして管理する想定です。以下のコマンドで初回コミットまで進められます
（**GitHubリポジトリの作成・`git push` はユーザーの明示的な許可なしに自動実行していません**）。

```bash
cd crowdworks-sales-assistant
git init
git add .
git commit -m "feat: クラウドワークス案件収集・管理ツールを追加"

# GitHub上に空のリポジトリを作成した後、以下でリモートを設定してpushしてください
git remote add origin https://github.com/<あなたのユーザー名>/crowdworks-sales-assistant.git
git branch -M main
git push -u origin main
```
